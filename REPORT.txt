Отчёт по второму этапу проекта Rail Ticket Harvester

Реализованные шаги:
- Добавлен режим покупки билетов через консольное приложение.
- Реализован ввод данных пассажира, email и платежной информации.
- Внедрена валидация ввода с возможностью повторных попыток без выхода из программы.
- Сделан запрос на оплату билета через API Rail Ninja с передачей данных в JSON формате.
- Обработан ответ API, возвращающий статус оплаты, который выводится пользователю.
- Обеспечена работа с сессиями и куки для взаимодействия с API.
- Улучшен консольный интерфейс для удобства пользователя.

Возникшие трудности:
- Проблемы с обработкой сессий и куки, необходимых для корректных запросов.
- Ошибки при формировании и валидации платежных данных (форматы даты, CVV).
- Необходимость сделать программу устойчивой к ошибкам ввода и повторному вводу данных.
- Трудности с асинхронностью и несколькими запросами к API для получения билетов и оплаты.

Затраченное время:
- Около 25-30 часов на реализацию покупки, обработку платежей, валидацию и отладку и рефакторинг.

Дополнительные замечания:
- В ходе работы было допущено много ошибок по невнимательности, что приводило к дополнительным затратам времени на исправление.
- Часто приходилось перепроверять форматы данных и логику обработки из-за пропущенных деталей.
- Тем не менее, все задачи были успешно выполнены и функционал полностью работает.
